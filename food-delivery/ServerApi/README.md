# ServerApi — серверный интерфейс и бизнес-логика

## Что находится в этой папке
Серверная часть проекта: API для мобильного приложения и веб-интерфейсов, бизнес-правила, работа с данными, интеграции и безопасность.

## Планируемые компоненты/модули
- **Auth & Users**: пользователи, роли (клиент/ресторан/курьер/админ), токены, права доступа
- **Restaurants**: рестораны, зоны доставки, графики работы
- **Menu**: категории, блюда, модификаторы, цены, стоп-лист
- **Orders**: создание заказа, статусы, отмены, перерасчёт, история
- **Payments**: создание платежа, подтверждения/вебхуки, возвраты (частично/полностью)
- **Delivery**: назначение курьера, расчёт стоимости/времени, трекинг (опционально)
- **Notifications**: пуши/смс/email (по необходимости), события статусов
- **Admin & Analytics**: отчёты, метрики, журналы действий (минимально на MVP)
- **Integrations**: платежный провайдер, карты/геокодинг (если потребуется)

## Взаимодействие с другими частями проекта
- **MobileApp** и **Web** — клиенты API (HTTP(S)).
- ServerApi — единый “источник правды”: хранит данные и применяет правила (что можно заказать, когда, по какой цене, какой статус допустим).
- Для обновлений в реальном времени ServerApi может отдавать события/стрим статусов (способ фиксируется архитектурно).
