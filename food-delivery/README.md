# FoodRush — приложение для доставки еды

## Идея проекта
FoodRush — сервис доставки еды, который объединяет клиентов, рестораны и курьеров в одном решении:
- клиент выбирает блюда, оплачивает и отслеживает заказ;
- ресторан принимает заказ и готовит;
- курьер получает доставку и довозит до клиента.

Цель проекта — обеспечить быстрый путь от “хочу есть” до “вот ваш заказ”, без магии, но с трекингом.

## Структура репозитория
- **MobileApp/** — мобильное приложение (клиентская часть: заказ, оплата, трекинг)
- **ServerApi/** — серверный API и бизнес-логика (пользователи, рестораны, меню, заказы, платежи, доставка)
- **Web/** — веб-сайт / админ-панели (витрина, управление меню, заказами, аналитика)

## Как части проекта связаны
- **MobileApp** и **Web** взаимодействуют с системой через **ServerApi** по HTTP(S) API.
- **ServerApi** предоставляет единый источник данных и правил: меню, цены, статусы заказов, авторизация, роли.
- Реальное время (статусы заказа/курьера) реализуется через механизмы push/WS/long-poll — решение фиксируется на этапе проектирования.
