# Web — веб-сайт и веб-интерфейсы

## Что находится в этой папке
Веб-часть проекта. В зависимости от выбранной стратегии сюда могут входить:
- публичный сайт (витрина, промо, условия, контакты);
- панели управления (для ресторана и/или админа);
- служебные интерфейсы (поддержка, управление заказами).

## Планируемые компоненты/модули
- **Landing / Public pages**: главная, условия, FAQ, контакты
- **Restaurant Dashboard**:
  - управление меню (блюда, цены, стоп-лист);
  - обработка заказов (принять/отклонить, статусы);
  - базовая аналитика (выручка/кол-во заказов).
- **Admin Panel** (минимально для MVP):
  - управление ресторанами и пользователями;
  - просмотр заказов и спорных ситуаций;
  - настройки справочников (если потребуется).
- **Courier View** (опционально):
  - список доставок, статусы, маршрут.

## Взаимодействие с другими частями проекта
- Web обращается к **ServerApi** для:
  - авторизации (роли: ресторан/админ/курьер);
  - управления меню и заказами;
  - получения отчётов и данных.
- Публичный контент может быть независимым, но данные ресторанов/меню для витрины (если решим показывать их на сайте) также берутся из **ServerApi**.
